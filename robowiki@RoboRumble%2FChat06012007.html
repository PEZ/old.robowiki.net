The smart algorithm works as follows:

  • Bots with less than 500 battles have the priority (now it was only MicroAspid).
  • Participants files and ratings files are checked every two hours. So any new bot should be downloaded in this time to the clients.
  • When a client uploads a battle, it also downloads the number of battles the bots have fought. So a new bot will never get more than 500 battles in priority (plus a maximum of 30 per client because the battles list is not recalculated once it is set up).
  • Because a battle has two bots, the other bots will fight against the new ones, and that's good because they probably haven't.
  • It is true also for minis/micros/nanos. First the a bot gets 500 battles in the general ranking, then 500 in the minis, 500 in the micros, and finally 500 in the nanos ranking (only if it belongs to the category).

It should speed up the updating of new bots. Next step will be to make the clients run the battles bots have pending against a given enemy. -- Albert

The smart algorithm works as follows:

  • Bots with less than 500 battles have the priority (now it was only MicroAspid).
  • Participants files and ratings files are checked every two hours. So any new bot should be downloaded in this time to the clients.
  • When a client uploads a battle, it also downloads the number of battles the bots have fought. So a new bot will never get more than 500 battles in priority (plus a maximum of 30 per client because the battles list is not recalculated once it is set up).
  • Because a battle has two bots, the other bots will fight against the new ones, and that's good because they probably haven't.
  • It is true also for minis/micros/nanos. First the a bot gets 500 battles in the general ranking, then 500 in the minis, 500 in the micros, and finally 500 in the nanos ranking (only if it belongs to the category).

It should speed up the updating of new bots. Next step will be to make the clients run the battles bots have pending against a given enemy. -- Albert
