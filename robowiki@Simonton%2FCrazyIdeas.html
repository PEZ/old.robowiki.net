It would be sweet if Robocode could run a JVM inside the JVM (do they have a JVM written in Java? That would be weird), to run each bot for a set maximum number of instructions per turn. Using a set amount of time is just so prone to interference. It would make robocode more suitable to pause & resume (maybe as a screen saver?), and could eliminate the unfair advantage bots like Toad have by bogging down the CPU during your turn, doing its own calculations & starving you of time for your own. And I would never have to wonder again how far above/below my own CPU constant my bots need to run to be safe on everyone else's machines.
