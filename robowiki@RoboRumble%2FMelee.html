Not sure, but it should be quite easy:

  • Create a class (or a method) that prepares the melee battles (this is a basically file basically where each line is a battle, with the bots separated by comas).
  • Change the BattlesRunner? class to deal with melee battles. I think that the only thing that should be changed is the part that reads the results from the table and stores them in a file. If the results are uploaded as "artificial" 1v1 pairings, then that's all you need to do.
  • In the server side, if you decide to change the number of rounds for melee, you will have to adjust the UploadedResults? servlet to filter the melee results using the new number of rounds (now is 35 by default, 10 for teams).

About rankings, if I understod Paul correctly, it's just taking each possible pair of competitiors and uploading them as a single 1v1 battle. For example:

  • 1rs: botA, score 5000
  • 2nd: botB, score 2500
  • 3rd: botC, score 1000

It will be uploaded like:

  • botA,5005,botB,2500
  • botA,5000,botC,1000
  • botB,2500,botC,1000
